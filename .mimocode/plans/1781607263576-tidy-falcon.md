# Carbon Agent â†’ Enterprise Operating System

## Goal

Transform Carbon Agent from a single-user document-reasoning script runner into an Enterprise Operating System for AI that can replace Claude Desktop and Cowork/Cork.

## Strategic pillars

### A) Replace Claude Desktop
1. System tray daemon + global hotkey with cross-app screen context.
2. Unified knowledge graph indexing documents, emails, calendars, PM tools, and databases.
3. Proactive background watchers with anomaly detection and user alerts.

### B) Replace Cowork / Cork
4. Multi-tenant architecture with role-based access control and project workspace isolation.
5. Human-in-the-loop workflow engine with approval gates in agent processes.
6. Shared team memory where agents keep consistent client context and hand off work.

## Clarified scope

- **Multi-tenancy is local-first, SaaS-later**: start with multiple user accounts sharing the same local SQLite database and Electron install, leaving network/tenant SaaS for a later phase.
- **Persistence first**: wire durable storage for memory tiers before adding extraction or UI.
- **Composition over rewrites**: reuse existing `AgenticMemorySystem`, `EnterpriseAgentHarness`, `AgentRuntime`, and IPC patterns.

## Phase order

| # | Phase | Pillors | Risk |
|---|-------|---------|------|
| 1 | Graph memory persistence + `AgenticMemorySystem` integration | A.2 foundation | Low |
| 2 | Episodic memory persistence + ingestion-driven extraction | A.2, A.3 seed | Low |
| 3 | Connector framework + proactive anomaly watchers | A.2, A.3 | Medium |
| 4 | Multi-tenancy schema + RBAC enforcement | B.4 | Medium |
| 5 | Tool permission enforcement + shared team memory | B.4, B.6 | Medium |
| 6 | Human-in-the-loop approvals (`supervisionMode: "confirm"`) | B.5 | Medium-High |
| 7 | Screen-context daemon (tray, global hotkey, capture) | A.1 | High |

## Phase 1: Graph Memory Persistence & Integration

### Goal
Promote `GraphMemory` from an in-process cache to a durable first-class tier with IPC endpoints.

### Key work
- `packages/local-store/src/sqlite.ts`
  - Add CRUD methods: `dbStoreGraphNode`, `dbGetGraphNode`, `dbFindGraphNodes`, `dbMergeGraphNode`, `dbStoreGraphEdge`, `dbFindGraphEdges`, `dbDeleteGraphNode`.
  - Indexes: `memory_graph_nodes(workspace_id, type)`, `memory_graph_edges(source_id, target_id, relation_type)`.
- `packages/local-store/src/graph.ts` (new): persistence helper wrapping DB graph methods.
- `packages/core-runtime/src/memory/graph.ts`: optional persistence callback + hydration.
- `packages/core-runtime/src/memory/system.ts`: integrate `GraphMemory` into lifecycle and public API.
- `packages/shared-schemas/src/index.ts`: add IPC schemas
  - `graph/list`, `graph/get`, `graph/createNode`, `graph/createEdge`, `graph/merge`, `graph/delete`, `graph/query`.
- `apps/desktop/src/ipc-handlers.ts`: add graph route handlers under `carbon-ipc`.
- `packages/local-store/src/migrations/01_graph_memory.sql` (new): idempotent index migration.
- Tests: round-trip persistence in `core-runtime` and `local-store`.

### Acceptance
- `pnpm -r typecheck` passes.
- Existing tests pass.
- New test: create nodes/edges, restart `AgenticMemorySystem`, and recall persisted data.
- IPC `graph/list` returns nodes for a workspace.

### Dependencies
None.

## Phase 2: Episodic Memory Persistence & Extraction

### Acceptance

- Episodic events survive restart.

- Tests cover extraction + episodic round-trip.

### Dependencies

  - Add nullable `tenant_id` and `user_id` columns to existing tables; default existing rows to a built-in tenant/admin.
- `packages/local-store/src/rbac.ts` (new): `hasPermission`, `listUserRoles`, `canAccessWorkspace`, `assignRole`, `ensureTenantAdmin`.
- `apps/desktop/src/auth.ts` (new): local auth/session service.
- `apps/desktop/src/ipc-handlers.ts`: add auth gate before `carbon-ipc` switch; add `auth/login`, `auth/logout`, `admin/*` handlers; enforce workspace membership.
- `apps/desktop/src/db-context.ts` + `apps/desktop/src/secure-storage.ts`: support per-user/tenant encryption contexts.
- `apps/desktop/src/main.ts`: initialize auth/session; seed default tenant + admin on first run.
- `packages/shared-schemas/src/index.ts`: add auth/admin IPC schemas.
- `apps/desktop/src/renderer/views/workspaces-view.ts`: filter workspace list by membership.
- `packages/local-store/src/migrations/04_multitenancy.sql` (new).

### Acceptance
- Existing tests pass with backward-compatible nullable columns.
- New tests verify cross-user access denial and admin override.
- Workspace list is membership-filtered.

### Dependencies
- `bcryptjs` or `argon2` for password hashing.
- `uuid` if not already present.

- `packages/core-runtime/src/harness.ts` and `packages/core-runtime/src/browser-orchestration.ts`: add permission check hook before tool/harness execution.
- `packages/core-runtime/src/memory/system.ts` + `skills/` APIs: filter by workspace/tenant and respect membership.
- `apps/desktop/src/ipc-handlers.ts`: gate skill/memory/graph routes by membership.
- `apps/desktop/src/preload.ts` + `global.d.ts`: expose permission/session context if renderer needs it.

### Acceptance
- A tool marked `permissions: ["skills:admin"]` is blocked for a viewer user and allowed for an admin.
- Workspace members see the same graph/memory/skills; non-members are denied.
- Existing supervision modes continue to work.

## Phase 6: Human-in-the-Loop Approvals

### Goal
Turn `supervisionMode: "confirm"` into a real pause/resume flow.

### Key work
- `packages/shared-schemas/src/index.ts`: add `session/approve`, `session/reject`, `session/listPendingApprovals` schemas.
- `packages/core-runtime/src/harness/approval-coordinator.ts` (new): manages pending approvals by correlation ID, timeouts, resume/reject callbacks.
- `packages/core-runtime/src/harness.ts` + `packages/core-runtime/src/browser-orchestration.ts`: when `supervisionMode === "confirm"`, emit `judgment_requested` and await coordinator Promise; emit `output_approved`/`output_rejected` accordingly.
- `apps/desktop/src/ipc-handlers.ts`: add approval endpoints that resolve coordinator promises.
- `apps/desktop/src/desktop-events.ts`: push `approval-requested` to renderer.
- `apps/desktop/src/preload.ts` + `global.d.ts`: add approval APIs and `onApprovalRequested` listener.
- Renderer UI: approval card with approve/reject/context.
- `packages/local-store/src/migrations/05_approvals.sql` (new, optional persistence of pending approvals).

### Acceptance
- Unit test confirms `"confirm"` pauses until approve/reject.
- Renderer can approve/reject and the run continues or halts.
- `"watch"` mode unchanged.
- Timeout automatically rejects.

### Dependencies
None.

## Phase 7: Screen-Context Daemon

### Goal
Add system tray, global hotkey, active-window metadata, and screenshot capture feeding the existing event bus.

### Key work

### Acceptance

## Cross-cutting concerns

| Concern | Approach |
|---------|----------|
| Security | Encrypt connector/OAuth secrets via `crypto.ts`/`secure-storage.ts`; enforce least-privilege roles; log security events to episodic memory; sanitize connector SQL and IPC inputs. |
| Migrations | One numbered idempotent SQL migration per phase under `packages/local-store/src/migrations/`. Run a migration runner on desktop startup. Wrap schema changes in transactions. Never drop `workspace_id`. |
| Schema versioning | Add `schema_version` table; desktop startup applies missing migrations before accepting IPC. |
| Native modules | Configure `electron-builder` to rebuild native deps; list them in `onlyBuiltDependencies`; add macOS codesign/notarization entitlements for screen capture and accessibility. |
| Tests | Each phase adds package unit tests and at least one desktop Playwright e2e test for user-visible behavior. Connector tests are mocked. Migration tests use a fresh temp DB. |
| Performance | Batch graph writes; throttle screen capture; incremental connector sync with stored cursor state; run memory consolidation off the main thread where possible. |
| Privacy | Screen capture is opt-in and configurable; implement blur/redaction; honor data retention via existing memory decay/consolidation. |

## Deferred scope

- SSO/OAuth/enterprise identity (phase 4 uses local auth; add later).
- Real-time collaborative cursors / websockets (use polling/SSE initially).
- ML-based anomaly detection beyond threshold rules.
- Mobile clients.
- Billing, quotas, metering.
- Connector marketplace / plugin SDK.
- Background audio transcription.
- Custom OS-level global shortcut hooks beyond Electron + `node-global-key-listener`.
